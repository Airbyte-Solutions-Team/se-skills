## Summary

Closes the UI/UX improvement phase and completes SEC-005 by moving the Anthropic API key out of plaintext `~/.mcp/*.env` files and into the OS keyring.

What changed:
- `_output_validation_status` no longer treats `validation_status: "unvalidated"` (normal for skills without an output schema, e.g. `deal-assessment`) as a validation attention item. Unvalidated outputs still appear in the review queue when they lack human feedback, but they do not inflate the "Needs attention" count with an `incomplete` validation label.
- `IMPLEMENTATION-PLAN.md` is updated: UX-001/002/004/006/007/008/009 and Phase 4 are marked `Completed`; deferred UX ideas are recorded; Phase 2 is marked complete with SEC-005 done.
- The quick ask-bar path now reads `ANTHROPIC_API_KEY` from the OS keyring via `keyring.get_password("se-skills", "ANTHROPIC_API_KEY")`, falling back to the `ANTHROPIC_API_KEY` environment variable. The previous `~/.mcp/*.env` parser is removed.
- `_anthropic_api_key()` centralizes the lookup and is used by the two ask-bar endpoints and `/api/ai-status`.
- `keyring` is added to `pyproject.toml` dev extras and to the PEP 723 inline deps in `webapp/app.py`.
- `webapp/README.md` now instructs users to store the key with `keyring set` (env var fallback) and to delete old `~/.mcp/*.env` files.
- New tests cover env/keyring precedence and fallback.
